## 1. 文档定位

1. 本文档是当前项目的长期规则来源。
2. 后续新建 PRD run、模块开发、review 默认读取项目内固定路径 `docs/project/project-conventions.md`。
3. 如需整体修改主题、组件体系、工程基线，进入初始化变更流程，而不是在普通 PRD 中临时重写。

## 2. 设计风格总则

1. 固定风格方案：Flat Design + Minimalism + AI-Native UI（轻量叠加）
2. 风格来源基线：AI视频生成系统商家端初始化基线 - experience_platform
3. 固定主题模式：浅色主题为正式默认模式，页面和组件 token 先围绕高可读后台场景定义。
4. 暗色模式策略：仅保留后续扩展能力，当前不设计双主题并行差异稿
5. 固定信息密度：中等信息密度，表格、筛选、表单、媒体预览和任务状态需在同一页面共存时优先保证扫描效率。
6. 后续页面默认继承以上规则，不再各模块自行重定一套视觉语言。

## 3. 导航与应用骨架规则

1. 固定导航原则：左侧主导航承载核心模块，顶部上下文区承载租户信息、页面级操作、搜索和状态入口；页面内部不再自建第二套主导航。
2. 固定 app shell：
   1. 左侧主导航 + 顶部上下文区 + 统一页面内容容器的三段式后台骨架。左侧固定工作台、生成任务、素材、合规、账户等一级入口；顶部统一承载租户上下文、全局搜索、通知和页面级主操作；正文区统一使用页头、内容区、侧边详情抽屉的继承式结构，不允许每个业务模块重新定义壳层。
3. 页面内部不再自建第二套主导航。
4. 全局壳层、页头容器和正文骨架默认继承统一应用骨架。

## 4. Theme / Token 规则

### 4.1 Spacing

- `space-1` = `4px`：图标与文字、标签内微间距、状态点与文案之间的最小补白
- `space-2` = `8px`：输入框内边距、小按钮内容补白、表格操作项之间的标准紧凑间距
- `space-3` = `12px`：表单项标题与控件、卡片内部小区块、预览信息块的默认垂直间距
- `space-4` = `16px`：卡片内标准内边距、筛选栏单元、列表页工具区常规间距
- `space-5` = `24px`：页面区块、工作台卡片组、结果区与侧栏之间的主层级间距
- `space-6` = `32px`：页面主体容器默认内边距，适合 PC 后台常规内容区
- `space-7` = `48px`：大型工作区切换、空状态或分段页面的增强留白
- `space-8` = `64px`：仅用于首页头部概览区或宽屏工作台的大块分段，不进入普通表单和列表微布局

### 4.2 Radius

- `radius-sm` = `8px`：输入框、标签、轻量按钮和表格内状态块，保持稳健而非玩具化圆角
- `radius-md` = `12px`：标准卡片、筛选面板、抽屉区块和模态内容容器
- `radius-lg` = `16px`：首页工作台概览卡、媒体预览容器和大分栏面板

### 4.3 Shadow

- `shadow-sm` = `0 1px 2px rgba(15,23,42,0.06), 0 8px 20px rgba(15,23,42,0.04)`：默认卡片与浮层的基础抬升，强调轻层级而非重阴影
- `shadow-md` = `0 12px 30px rgba(15,23,42,0.10)`：抽屉、弹窗、结果预览浮层和首页重点概览卡
- `shadow-focus` = `0 0 0 3px rgba(37,99,235,0.16)`：聚焦与选中态统一使用语义外发光，不单独发明局部描边效果

### 4.4 Typography

- `text-xs` = `12/18`：辅助说明、时间信息、状态补充文案和次级标签
- `text-sm` = `14/22`：表单控件、表格正文、筛选栏和大多数后台正文信息
- `text-md` = `16/24`：页面正文强调段、详情说明、关键操作说明区
- `text-lg` = `20/28`：页面标题、工作台区块标题、结果页主分区标题
- `text-xl` = `24/32`：首页概览大标题和关键数据区块标题，不用于普通业务页正文

### 4.5 Color Roles

- `surface` = `#F8FAFC`：页面背景、主内容区底色和非强调型后台底盘
- `surface-raised` = `#FFFFFF`：卡片、筛选面板、抽屉主体和弹窗内容层
- `surface-muted` = `#F1F5F9`：次级分区、表格头部、上传占位区和只读信息块
- `text-primary` = `#0F172A`：页面标题、主数据、关键正文和高优先级操作文案
- `text-secondary` = `#475569`：描述文案、提示信息、次级导航和表单说明
- `border-default` = `#E2E8F0`：输入框、卡片、表格分隔线和容器边界
- `accent` = `#2563EB`：主操作、链接、选中态和 AI 任务流中的关键行动点
- `accent-soft` = `#DBEAFE`：选中背景、轻提示底色和信息型状态底板
- `success` = `#16A34A`：生成成功、通过状态、可用余额或正常运行反馈
- `warning` = `#D97706`：额度提醒、处理中提示、待完善和风险提示
- `danger` = `#DC2626`：失败状态、删除动作、合规未通过和高风险提醒

## 5. 页面实现规则

1. 首页工作台采用“关键指标条 + 待处理事项 + 最近任务/异常提醒”混合布局，指标卡可加入轻媒体预览，但主目标仍是快速扫描和进入任务。
2. 生成任务与素材列表页默认采用“页头操作区 + 筛选栏 + 主列表/卡片区 + 右侧详情抽屉”模式，支持高频筛选、状态查看和单条进入。
3. AI 生成结果页采用“参数配置区 / 结果预览区 / 历史版本或操作记录区”的稳定分栏，结果预览是焦点，但参数和状态说明必须与之同屏可读。
4. 上传与创建流程采用分步面板或抽屉式流程，不把大表单直接堆成超长单页；上传态、处理中、失败态、成功态要复用统一反馈容器。
5. 账户、账单、消耗和通知类页面以列表和详情块为主，视觉重点低于创作工作区，不单独发明一套“财务后台”风格。
6. 页面实现先复用统一容器和页面模式，再补模块私有组合。
7. 不为了单个模块重写导航、全局壳层或页面级布局规则。

## 6. 组件风格与封装规则

1. 优先复用 PageHeader、PageSection、Card、KpiCard、FilterBar、DataTable、DetailDrawer、FieldGroup、StatusBadge、EmptyState、UploadDropzone 等基础容器，业务页面只做组合，不重写底层样式语言。
2. 媒体预览卡与普通信息卡共享同一圆角、边框和阴影体系，差异只通过比例、封面区和状态层表达，不额外定义第二套卡片视觉。
3. 按钮、标签、提示、进度、通知和审计状态统一映射到语义色角色，所有成功/警告/失败/处理中反馈都从统一状态组件扩展。
4. 表单、筛选和参数面板统一使用两层层级：容器层负责分区，字段层负责对齐；禁止在单页里混用自由摆放字段和自定义块状表单。
5. 需要 AI 产品感时，优先通过分栏结构、进度反馈、结果焦点和轻量动态强调实现，不把聊天气泡、霓虹渐变或大面积玻璃拟态引入主工作区。

## 7. 禁止事项

1. 页面内直接写裸 spacing、radius、color、shadow 值，绕开 token 和语义角色体系。
2. 在某个业务模块局部重新定义卡片、按钮、标签、表格或抽屉样式，导致同一后台出现两套基础组件语言。
3. 把营销页 Hero、大面积渐变插画、强玻璃效果或沉浸式深色工作区直接带入普通商家后台页面。
4. 把 AI 视频生成页面做成聊天产品样式，或让结果区完全脱离参数区与状态区，导致任务信息不可追踪。
5. 为了追求信息量而压缩到极端紧凑密度，或反过来用过度留白牺牲商家后台扫描效率。

## 8. 使用方式

1. PRD 阶段先读取本文档，再开始页面模式、组件复用和视觉实现相关设计。
2. 设计/开发阶段持续将本文档作为项目级视觉与交互规则输入。
3. 如需补充新的项目级视觉规则，应先判断是否属于当前项目长期特征，再更新本文档。
