# PRD Bootstrap Context Template

这份模板用于生成交给下一轮 `prd` 的基础 PRD 上下文文件。

目标：

1. 把 `init-01` 到 `init-04` 已确认的项目基础结论整理成下一轮 `prd` 的起始输入。
2. 让下一轮 `prd` 先围绕登录、账号、租户、权限这类基础模块展开。
3. 不提前带入任何具体业务功能前提，避免后续 AI 把业务方向误当成已确认范围。
4. 全文尽量使用清单和层级结构，不写初始化总结散文。

建议结构：

## 1. 文档目标

- 说明本文档是 init 流程交给下一轮 `prd` 的基础 PRD 输入。
- 说明本文档先固定项目概况和基础模块方向，不提前带入具体业务功能。
- 说明生成方式是：脚本先产骨架，AI 再补强，reviewer 再检查边界。

## 2. 项目概况

- 简要说明项目是什么、服务谁、默认地区与语言是什么。
- 只保留项目级概况，不写业务模块拆解。

## 3. 已确认基础前提

- 列出登录方式、账号标识、账号模型、租户模型、权限模型等基础结论。
- 每条都写成后续 `prd` 可直接继承的需求前提。

## 4. 基础模块需求

- 先围绕登录/认证、账号/成员、租户上下文、权限/访问控制等基础模块组织。
- 每个模块给出一句目标，再列 3-5 条直接需求清单。
- 需求写法应直接说明“系统应具备什么”，不要写成“下一步需要明确什么”。

## 5. 本轮 PRD 关注点

- 说明下一轮 `prd` 应重点细化哪些基础功能问题。
- 不写任何具体业务功能前提。

## 6. 备注

- 如项目级前提整体变化，应回到初始化变更流程。
- 项目级长期规则仍以 `docs/project/project-conventions.md` 为准。
- reviewer 应重点检查：是否混入具体业务前提、是否遗漏基础组件/能力组件边界、是否仍然像基础 PRD 而不是流程说明。
