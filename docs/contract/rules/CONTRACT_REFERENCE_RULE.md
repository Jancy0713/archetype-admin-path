# Contract Reference Rule

## 目标

这份规则用于约束 `contract` 在按 batch 或按模块拆分后，如何引用其他已存在的正式 contract 定义。

它要解决的问题是：

- 哪些内容应该复用前序 frozen contract
- 哪些内容应该由当前 batch 新定义
- 哪些引用是合法的
- 哪些引用是不允许的

## 核心原则

1. 只引用正式 frozen contract，不引用未冻结的中间状态。
2. 能复用已有共享定义的，不重复定义。
3. 引用关系必须可定位、可追踪、可脚本查找。
4. 当前 batch 只对自己负责的新增定义建模，不随意重写他人边界。
5. 一旦某定义已被其他 batch 依赖，破坏性修改不能隐式覆盖。

## 当前正式引用主键

当前建议先统一：

- 引用目标以 `contract_id` 作为正式主键

在 MVP 阶段，默认可采用：

- `contract_id = batch_id`

后续如果 `module_id` 与 `batch_id` 不再一一对应，再单独演进映射规则，但当前不要同时并行使用多套主键。

## 什么情况下应该引用，而不是新定义

以下场景优先引用：

1. 共享资源已在前序 frozen contract 中正式定义。
2. 状态或枚举已在共享 contract 中稳定存在。
3. 某个动作语义已由前序 contract 明确约束。
4. 当前 batch 只是消费已有资源，不是重新定义其 canonical 协议。

## 什么情况下可以在当前 batch 新定义

可以新定义的前提通常是：

1. 当前内容确实属于本批职责范围。
2. 前序 frozen contract 中不存在稳定定义。
3. 当前 batch 是这个资源、动作、枚举或 view 的正式归属方。
4. 当前定义不会与已有共享定义产生冲突。

## 不允许的引用方式

不允许：

1. 引用其他 flow 尚未 release 的中间产物。
2. 复制已有共享定义后在本批静默改写。
3. 用自然语言模糊表达“依赖某模块”，但不给可定位引用点。
4. 在没有说明来源的情况下直接复写前序 contract 里的字段语义。

## 当前推荐的引用层次

建议至少区分两层：

### 1. 模块级依赖

用于说明：

- 当前 batch 或当前 contract 依赖哪些模块级 contract

例如：

- `depends_on_modules`

### 2. 具体符号级引用

用于说明：

- 当前 batch 实际引用了哪个资源、枚举、动作或共享类型

例如未来可抽象为：

- `type_ref`
- `enum_ref`
- `action_ref`
- `view_ref`

当前阶段先不定最终字段名，但必须先定这个分层原则。

## 当前 batch 必须显式说明的引用信息

如果当前 batch 依赖其他 contract，至少应说明：

1. 依赖的是哪个前序 frozen contract
2. 依赖的是哪个共享对象或语义块
3. 当前是复用它，还是在其基础上扩展 view 消费面
4. 当前 batch 自己新增的部分与引用部分的边界

## 对 `domain_mapping` 的要求

`domain_mapping` 阶段必须先把引用计划标出来。

至少要回答：

1. 哪些定义来自前序 contract
2. 哪些定义由本批新增
3. 哪些引用必须在 `contract_spec` 中正式固化

如果这一步不做，`contract_spec` 很容易：

- 误重定义共享对象
- 漏掉依赖关系
- 让下游无法追踪来源

## 对 `contract_spec` 的要求

`contract_spec` 中的引用必须达到：

1. 可定位
2. 可解释
3. 可供下游消费

也就是说，不能只写：

- “复用用户模块定义”

而应至少达到“下游知道你到底复用了哪块正式定义”的程度。

## 对 reviewer 的要求

reviewer 应重点检查：

1. 是否本应引用却被重新定义
2. 是否错误引用了未冻结结果
3. 是否引用关系不清，导致来源不可追踪
4. 是否把共享对象和本批新增定义混在一起

## 放行条件

只有当下面条件成立时，引用关系才可视为通过：

1. 所有关键外部依赖都来自 frozen contract
2. 共享定义与新增定义边界清楚
3. 当前 batch 的新增定义不会与已有共享定义冲突
4. 下游可以根据当前 contract 稳定查找到依赖来源

## 常见错误

1. 为了省事直接复制前序 contract 定义
2. 在当前 batch 偷偷改写共享定义
3. 只写“依赖某模块”，不给来源锚点
4. 用未冻结的 run 过程文件当作正式引用目标
