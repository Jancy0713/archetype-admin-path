# Claude Code PM Course 阅读摘要

## 来源

- GitHub: [carlvellotti/claude-code-pm-course](https://github.com/carlvellotti/claude-code-pm-course)

## 我们关注到的内容

根据仓库 README，这个项目的定位是：

- 一个面向 Product Manager 的交互式课程
- 目标是教 PM 如何把 Claude Code 用在日常工作中
- 重点覆盖：
  - PRD 编写
  - 研究整理
  - 多 agent 协作
  - 自定义 sub-agent
  - 项目记忆

## 和我们当前问题最相关的点

### 1. AI 是思考伙伴，不只是自动执行器

这个仓库强调的是：

- Claude Code 不是单纯帮 PM 自动生成文档
- 而是帮助 PM 做思考、组织、反馈和迭代

这和我们当前方向是一致的。

因为我们现在要做的不是“自动写 PRD”，而是：

- 先通过 AI 补问
- 再把需求结构化
- 再进入后续 contract 和生成链路

### 2. 课程式拆分很有参考价值

它把能力拆成：

- 基础操作
- agent 使用
- 项目记忆
- PRD 场景
- 策略分析

这提醒我们一件事：

我们自己的工作流也不应该试图靠一个超长总文档解决全部问题，而应该按步骤拆成可复用模板。

### 3. Project Memory 很重要

README 明确提到使用 `CLAUDE.md` 作为上下文和说明承载。

对我们而言，对应的启发是：

- “拆解 PRD”过程中沉淀出的结构，不应该只停留在聊天记录
- 应该转成项目内可复用文档和模板

## 对我们流程的启发

这个仓库对我们最直接的启发，不是它的具体命令，而是它的角色定位：

- PM 不需要一开始就下实现命令
- 先让 AI 帮你把问题想清楚
- 再把中间产物固定下来

所以在我们的体系里，这个参考更像是：

- “为什么要做需求补充提问”
- “为什么 PRD 拆解应该成为独立步骤”

而不是直接照搬成具体技能设计。

## 当前结论

对我们最有价值的部分：

1. 把 AI 当成 PRD 和策略思考伙伴
2. 用分步骤教学思路来设计我们自己的工作流
3. 强化中间产物沉淀，而不是依赖会话上下文
