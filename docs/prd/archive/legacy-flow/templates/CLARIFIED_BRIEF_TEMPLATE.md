# 澄清版 Brief 模板

## 使用方式

当 `需求补充提问` 完成后，用这份模板输出一个可继续进入 PRD 拆解的中间文档。

这个 brief 不是最终 PRD，也不是实现方案。它的作用是：

- 固定当前已经确认的共识
- 把仍然缺失的关键决策显式列出来
- 作为后续 PRD 拆解、contract 设计的统一输入
