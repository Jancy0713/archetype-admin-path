# 版本规则

当前仓库采用轻量语义化版本：

- `MAJOR.MINOR.PATCH`

当前版本：

- `1.0.0`

## 规则

### `MAJOR`

只有当流程模型、目录结构或产物协议发生不兼容变化时才升级。

例如：

- `init/prd` 产物格式整体重构
- 运行目录结构变化，旧 run 不再兼容
- 核心脚本参数不兼容

### `MINOR`

当流程能力有明显增强，但兼容旧流程时升级。

例如：

- 新增自动化能力
- 新增 reviewer 策略
- 新增通用脚本
- 新增正式模板类型

### `PATCH`

当只是修正文档、提示词、脚本细节或测试流程时升级。

例如：

- 修改 prompt 规则
- 调整 run 初始化逻辑
- 修复进度板更新
- 增加小的命名约束

## 你当前的建议用法

### 首次正式提交

- 当前仓库状态作为 `1.0.0`

### 首轮真实测试

- run 命名使用：`init-1.0.0`

### 首轮测试后有调整

如果只是流程细节、prompt、脚本小修：

- 升到 `1.0.1`

如果是明显新增能力但不破坏原结构：

- 升到 `1.1.0`

如果流程协议或目录结构不兼容：

- 升到 `2.0.0`

## run 与仓库版本的关系

建议 run 名称直接带当前仓库版本。

例如：

- `init-1.0.0`
- `prd-1.0.1`

这样你一眼就能知道：

- 这轮测试基于哪个仓库版本
- 测试结果对应哪个流程版本
